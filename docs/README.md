# Документация Task Launcher

Этот каталог — постоянный источник продуктового и технического контекста между задачами Codex. Перед работой нужно прочитать документы, перечисленные в корневом [AGENTS.md](../AGENTS.md), и проверить открытые QA-записи.

| Документ                                               | Назначение                                                                 | Когда читать                                                             | Когда обновлять                                              | Тип                                               |
| ------------------------------------------------------ | -------------------------------------------------------------------------- | ------------------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------- |
| [product/current-scope.md](./product/current-scope.md) | Действующее продуктовое поведение и границы текущей реализации             | Перед любой продуктовой, UX или кодовой задачей                          | При каждом согласованном изменении поведения                 | Текущее состояние, канонический источник          |
| [product/decisions.md](./product/decisions.md)         | Принятые решения, их причины, последствия и статус                         | Перед изменением существующего поведения или архитектурной границы       | При принятии, замене или отклонении решения                  | Текущее состояние и журнал решений                |
| [qa/findings.md](./qa/findings.md)                     | Ошибки ручной и автоматической проверки, регрессии и связь с исправлениями | Перед исправлением интерфейса и перед завершением задачи                 | При обнаружении, исправлении или повторной проверке проблемы | QA-история; открытые записи задают текущую работу |
| [architecture.md](./architecture.md)                   | Устойчивые технические границы и направления зависимостей                  | Перед изменением интеграций, хранения, статусов, файлов или безопасности | Когда меняется архитектурное решение                         | Текущее устойчивое состояние                      |
| [roadmap.md](./roadmap.md)                             | Границы текущего и следующих milestones                                    | Перед планированием объема задачи                                        | Когда функция переносится между этапами                      | Текущие границы этапов                            |

Дополнительные документы:

- [README.md](../README.md) — краткий обзор и локальный запуск, не источник детальных продуктовых требований;
- [PLAN.md](../PLAN.md) — краткий рабочий план текущего milestone, не журнал решений;
- [AGENTS.md](../AGENTS.md) — короткая карта обязательных правил работы с репозиторием.

## Статусы

- В `decisions.md` действует только решение со статусом `Active`. `Superseded` сохраняется для истории и не задает текущее поведение.
- В `findings.md` запись `Open` остается обязательной к учету. `Fixed` означает, что код изменен и связанный регрессионный тест успешно выполнен.
- Если документы расходятся, применяется приоритет источников из [AGENTS.md](../AGENTS.md).
