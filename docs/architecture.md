# Архитектурные границы

Документ фиксирует устойчивые технические решения. Детали текущего поведения находятся в [product/current-scope.md](./product/current-scope.md), а этапы реализации — в [roadmap.md](./roadmap.md).

## Приложение

- Модульный монолит на Next.js App Router.
- TypeScript работает в strict-режиме.
- UI строится как mobile-first PWA для desktop, iPhone и Android.
- Server Components используются по умолчанию; Client Components добавляются только для интерактивности.
- Входные данные форм, environment и внешние ответы валидируются Zod-схемами.

## Направление зависимостей

```text
UI -> server action/use case -> repository/integration interface -> mock или live adapter
```

- UI не импортирует credentials и не зависит от REST-полей Bitrix24.
- `Bitrix24Client` является server-only границей интеграции.
- Mock- и live-адаптеры Bitrix24 являются разными реализациями одного контракта. В текущем milestone существует mock; live-адаптер добавляется позднее.
- Все внешние ответы преобразуются во внутренние DTO и нормализованные безопасные ошибки.

## Проекты и хранение

- Доступ к проектам проходит через `ProjectRepository`.
- Mock-реализация `ProjectRepository` входит в текущий milestone и проверена regression-тестами QA-010.
- Будущая Supabase-реализация подключается без изменения UI и use cases.
- Supabase Auth, Postgres, RLS и постоянное хранение входят в отдельный milestone.

## Статусы

- Технический результат отправки и доменный статус задачи являются разными понятиями.
- Значения Bitrix24 преобразуются во внутренний доменный статус `new`, `in_progress`, `completed` или `unknown`.
- UI отображает локализованное представление доменного статуса и не использует REST-константы напрямую.
- Live-синхронизация статусов добавляется вместе с Bitrix24-адаптером.
- Переход foundation с технических статусов на доменную модель выполнен в QA-008.

## Файлы

- Выбор и загрузка файлов проходят через серверную абстракцию.
- Mock-реализация сохраняет только безопасные метаданные файла.
- Бинарные данные не попадают в sanitized payload, историю операций и логи.
- Live-адаптер загрузки в Bitrix24 добавляется в интеграционном milestone.
- Интерфейс выбора файлов и mock-граница безопасных метаданных добавлены в QA-014.

## Безопасность

- Webhook, service credentials и другие секреты остаются только в server runtime.
- Секреты не попадают во frontend bundle, документацию, fixtures, пользовательские ошибки и логи.
- Server Actions и закрытые Server Components повторно проверяют identity и роль.
- Idempotency предотвращает случайную двойную отправку. Timeout не запускает автоматический повтор.

## Учет времени

- UI и внутренний create DTO не управляют учетом времени Bitrix24.
- Учет времени включает существующая автоматизация портала.
- Поля оценки и управляющий флаг учета времени не должны возвращаться без нового активного продуктового решения.
