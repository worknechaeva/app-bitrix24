# Текущий продуктовый scope

Этот документ фиксирует только действующее требуемое поведение Task Launcher. Это не хронология обсуждений. Расхождения между требованиями и текущим кодом перечислены как открытые записи в [QA-журнале](../qa/findings.md).

## Формат продукта

- Внутреннее адаптивное PWA для одного облачного портала Bitrix24.
- Интерфейс проектируется mobile-first и поддерживает desktop, iPhone и Android.
- Пользовательский интерфейс русскоязычный.
- Текущий milestone работает через development-only mock без внешних ресурсов.
- Supabase и настоящий Bitrix24 в текущей пачке изменений не подключаются.

## Главная страница

- Приветствие «Добрый день» отсутствует.
- Основное действие — кнопка создания задачи.
- Есть компактный блок последних задач.
- Ссылка «Все задачи» находится в заголовке блока последних задач и визуально согласована с заголовком.
- Недавний проект открывает форму создания задачи с уже выбранным проектом.
- Мобильная навигация остается доступной и не перекрывается аватаром или плавающими элементами.

## Создание задачи

Обязательные поля:

- проект;
- название задачи.

Необязательные поля:

- ответственный;
- срок;
- текст задачи;
- дополнительные теги;
- прикрепляемые файлы.

Поля и настройки, которых в форме быть не должно:

- приоритет;
- оценка в часах;
- управление учетом времени.

### Срок

- По умолчанию срок пустой.
- Пользователь может выбрать дату.
- Выбранную дату можно полностью очистить.
- Пустой срок не передается в интеграционный payload.

### Файлы

- На текущем этапе файлы выбираются через интерфейс.
- В mock-хранилище и истории сохраняются только безопасные метаданные файла: имя, MIME-тип и размер.
- Бинарные данные не входят в sanitized payload, историю и логи.
- Реальная загрузка файлов в Bitrix24 отложена до интеграционного milestone.

## Timeout и повторная отправка

- Автоматический повтор после timeout запрещен.
- Форма после timeout не блокируется навсегда.
- Пользователь получает понятное предупреждение и может вручную повторить отправку после проверки портала.
- Каждая ручная повторная попытка получает новый idempotency key.

## История

- История отображается компактным списком без повторяющейся декоративной иконки в каждой строке.
- Для записи отображаются задача, проект, ответственный, Bitrix24 ID и доменный статус.
- Доступен фильтр по проекту.
- Внутренняя модель поддерживает статусы `new`, `in_progress`, `completed` и `unknown`.
- До live-интеграции mock-данные используют доменные статусы без синхронизации с порталом.
- Получение и обновление live-статусов отложено до подключения Bitrix24.

## Проекты

- Администратор может создавать, редактировать, включать и выключать mock-проекты.
- Редактор может использовать активные проекты, но не может управлять ими.
- Проект содержит:
  - название;
  - адрес сайта;
  - ID группы Bitrix24;
  - название группы Bitrix24;
  - обязательный тег;
  - ответственного по умолчанию;
  - признак активности.
- У проекта нет настроек приоритета и учета времени.
- В текущем milestone изменения хранятся в mock-репозитории; постоянное хранение появится после подключения Supabase.

## Пока не подключено

- настоящий Bitrix24;
- Supabase;
- RLS;
- production-авторизация;
- live-статусы;
- реальная загрузка файлов;
- Vercel;
- офлайн-режим;
- push-уведомления.
